# سياسة الأمان — Security Policy

## الإصدارات المدعومة

| الإصدار | الدعم |
|:---:|:---:|
| 2.5.x | ✅ مدعوم بالكامل |
| 2.0.x | ⚠️ إصلاحات أمنية فقط |
| < 2.0 | ❌ غير مدعوم |

## الإبلاغ عن ثغرة أمنية

إذا اكتشفت ثغرة أمنية، **لا تفتح Issue عام**.

بدلاً من ذلك:

1. **أرسل بريد إلكتروني** إلى المطور عبر GitHub
2. **وصّف الثغرة** بالتفصيل مع خطوات إعادة الإنتاج
3. **انتظر الرد** — سنرد خلال 48 ساعة كحد أقصى

### ما الذي يُعتبر ثغرة أمنية؟

- ✅ تسريب مفاتيح API
- ✅ XSS (Cross-Site Scripting)
- ✅ حقن كود ضار في الردود
- ✅ الوصول غير المصرح به للوحة الأدمن
- ✅ تسريب بيانات المستخدمين من Firebase

### ما الذي لا يُعتبر ثغرة أمنية؟

- ❌ مفتاح API ظاهر في الكود (مصمم كـ client-side app)
- ❌ المحادثات محفوظة في localStorage (متوقع وموثق)
- ❌ أخطاء في الواجهة لا تؤثر على البيانات

## أفضل الممارسات الأمنية

عند استخدام أو نشر LORD AI:

1. **لا تشارك مفتاح API الخاص بك** علناً
2. **استخدم مفتاح API مع حدود استخدام** (Rate Limits)
3. **فعّل قواعد أمان Firebase** إذا كنت تستخدم التحليلات
4. **حدّث المشروع** إلى آخر إصدار دائماً
